# ADR 0004：不做反爬绕过

## 状态

已接受

## 日期

2026-06-29

## 背景

Job Radar 只用于本地自用的公开岗位信息整理。项目不需要登录、不需要自动投递，也不应采集候选人隐私数据。

为了降低合规、账号和网站负载风险，需要明确禁止验证码绕过、登录墙绕过、代理池规避封禁等行为。

## 可选方案

### 方案 A：实现反爬绕过能力

优点：

* 可能提高部分网站覆盖率。

缺点：

* 合规风险高。
* 工程复杂度高。
* 容易影响网站正常服务。
* 与项目 MVP 目标不匹配。

### 方案 B：只访问公开、无需登录、无强反爬的来源

优点：

* 风险低。
* 行为边界清晰。
* 更容易测试和维护。
* 与公开 ATS 优先策略一致。

缺点：

* 覆盖面会受限。
* 遇到验证码、登录墙或强反爬时需要人工处理。

## 决策

采用方案 B：不做任何反爬绕过。

明确禁止：

* 绕过验证码。
* 绕过登录墙。
* 使用代理池规避封禁。
* 使用盗取或个人账户 Cookie、Token。
* 高频请求压垮网站。
* 自动投递岗位。
* 采集候选人个人隐私数据。

采集策略记录在：

```text
docs/specs/005-crawl-policy.md
```

## 后果

正面影响：

* 项目边界清晰。
* 降低合规和账号风险。
* CI 与 fixture 测试更可控。

负面影响：

* 部分来源无法自动采集。
* 需要通过 `manual_review` 保留人工检查入口。
