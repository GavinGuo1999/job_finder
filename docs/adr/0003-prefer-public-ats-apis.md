# ADR 0003：优先使用公开 ATS 岗位源

## 状态

已接受

## 日期

2026-06-29

## 背景

项目需要采集公开岗位信息。不同公司官网结构差异很大，而 Greenhouse、Lever、Ashby、SmartRecruiters 等 ATS 通常提供较稳定的公开岗位页面或数据接口。

优先支持公开 ATS 可以降低解析成本，提高字段质量，并减少对脆弱 HTML 结构的依赖。

## 可选方案

### 方案 A：优先解析公司官网 HTML

优点：

* 覆盖面广。
* 不依赖特定 ATS。

缺点：

* 页面结构差异大。
* 字段质量不稳定。
* 维护成本高。

### 方案 B：优先使用公开 ATS 岗位源

优点：

* 字段结构相对稳定。
* 更容易做 fixture 测试。
* 更容易标准化岗位 ID、地点、部门和详情 URL。
* 更符合 MVP 的可维护性目标。

缺点：

* 不能覆盖所有公司。
* 某些公司使用自建招聘系统，仍需兜底。

### 方案 C：优先使用第三方招聘平台

优点：

* 岗位数量可能更多。
* 搜索和筛选能力强。

缺点：

* 登录、反爬和合规风险更高。
* 字段可能包含平台加工信息。
* 不适合作为第一阶段默认来源。

## 决策

第一阶段优先使用公开 ATS 岗位源，支持顺序为：

1. Greenhouse
2. Lever
3. Ashby
4. SmartRecruiters
5. JSON-LD
6. 静态 HTML 兜底

具体适配器规格记录在：

```text
docs/specs/006-ats-adapters.md
```

## 后果

正面影响：

* 第一阶段更容易稳定交付。
* 适配器更容易用 fixture 验证。
* 字段标准化质量更高。

负面影响：

* 一些没有公开 ATS 的公司需要 `manual_review` 或静态 HTML 兜底。
* 覆盖率可能低于直接抓第三方招聘平台。
